Google Search Console + Sitemap
Fait apparaître les annonces NEXUS sur Google avant celles de Jiji
30 min · 0 FCFA
⚠️ Sans Search Console, Google ne connaît pas l'existence de nexus.sn. Les annonces ne remontent jamais dans les résultats, peu importe leur qualité.
1
Aller sur Google Search Console
Se connecter avec le compte Google associé à nexus.sn.
🔗 Ouvrir Search Console
2
Ajouter la propriété
Cliquer Ajouter une propriété → choisir Préfixe d'URL → saisir :
https://nexus.sn/
3
Vérifier via balise HTML (méthode recommandée)
Google vous donne une balise méta à coller dans le <head> de index.html :
<meta name="google-site-verification" content="VOTRE_CODE_ICI" />
💡 Dans index.html, chercher la ligne <!-- Canonical --> autour de la ligne 227 et coller juste au-dessus.
4
Déployer sur Cloudflare Pages + cliquer Vérifier
Après le déploiement, revenir sur Search Console → cliquer Vérifier. Le bandeau vert confirme la vérification.
5
Télécharger et soumettre le sitemap.xml
Dans l'app NEXUS : Admin → SEO → Télécharger sitemap.xml
Uploader le fichier sur Cloudflare Pages dans le dossier public/ ou via une route Cloudflare Function.

Puis dans Search Console : Sitemaps → Ajouter un sitemap
https://nexus.sn/sitemap.xml
✅ Une fois le sitemap soumis, Google commencera à indexer les annonces dans les 48-72h. Résultats SEO visibles en 2-4 semaines.

Domaine custom nexus.sn
Crédibilité locale + SEO sénégalais + canal marketing
~50 $/an
ℹ️ Le domaine .sn est géré par le NIC Sénégal. Il faut un NINEA ou un justificatif sénégalais. Alternative : passer par un revendeur agrégé (Afrihost, Gandi).
1
Option A — NIC Sénégal (direct, moins cher)
Formulaire de demande d'enregistrement de domaine .sn. Nécessite une pièce d'identité et un justificatif d'activité.
🔗 nic.sn
1
Option B — Gandi.net (plus rapide, interface simple)
Revendeur international agréé pour les .sn. Carte bancaire internationale acceptée. Délai : 24-48h.
🔗 Gandi .sn
2
Pointer le domaine vers Cloudflare
Chez votre registrar, mettre les nameservers Cloudflare :
NS1 : ada.ns.cloudflare.com NS2 : bob.ns.cloudflare.com (les vôtres apparaissent dans Cloudflare Dashboard → votre domaine → DNS)
3
Ajouter le domaine custom dans Cloudflare Pages
Cloudflare Pages → votre projet → Custom domains → Set up a custom domain → saisir nexus.sn.

Cloudflare crée automatiquement le certificat SSL Let's Encrypt (HTTPS gratuit).
4
Mettre à jour les URLs OAuth (Google + Supabase)
Après activation du domaine, mettre à jour ces URLs dans les consoles concernées :
Google OAuth → Authorized origins : https://nexus.sn Google OAuth → Redirect URI : https://[project].supabase.co/auth/v1/callback Supabase Auth → Site URL : https://nexus.sn Supabase Auth → Redirect URLs : https://nexus.sn/**

Image OG — Partages WhatsApp & Facebook
Preview visuelle sur chaque lien partagé — canal marketing principal
Quick win · 0 FCFA
✅ L'image nexus-og.png (1200×630) a été générée dans ce dossier. Elle est prête à uploader.
1
Uploader nexus-og.png dans Supabase Storage
Supabase Dashboard → Storage → bucket nexus-images → Upload files → sélectionner nexus-og.png.
Copier l'URL publique générée (format https://xxx.supabase.co/storage/v1/object/public/nexus-images/nexus-og.png).
2
Mettre à jour les balises OG dans index.html
Chercher nexus.sn/og-image.png (3 occurrences) et remplacer par l'URL Supabase :
<meta property="og:image" content="https://XXX.supabase.co/storage/v1/object/public/nexus-images/nexus-og.png"> <meta name="twitter:image" content="https://XXX.supabase.co/storage/v1/object/public/nexus-images/nexus-og.png">
3
Alternative : Cloudflare Pages public folder
Placer nexus-og.png dans le dossier public/ du repo. Elle sera servie à https://nexus.sn/og-image.png directement (URL déjà dans le code).
💡 Option la plus simple — pas besoin de modifier le code si le fichier est déjà nommé og-image.png.
4
Valider le rendu WhatsApp
Coller l'URL dans Facebook Sharing Debugger → cliquer Debug → vérifier que la preview s'affiche. Forcer le rechargement du cache si nécessaire.
🔗 Facebook Debugger 🔗 Twitter Card Validator

OAuth Google + GitHub — Connexion sociale
Réduit l'abandon à l'inscription de ~60% · Gratuit
1h · 0 FCFA
🔵 Partie A — Google OAuth
1
Créer un projet sur Google Cloud Console
Aller sur console.cloud.google.com → Nouveau projet → nommer NEXUS Market.
🔗 Google Cloud Console
2
Activer l'API Google+ / Identity
APIs et services → Bibliothèque → chercher Google Identity Toolkit API → Activer.
3
Créer les identifiants OAuth 2.0
APIs et services → Identifiants → + Créer des identifiants → ID client OAuth → Application Web.

Remplir les champs :
Origines JavaScript autorisées : https://nexus.sn https://nexus-market-asb.pages.dev URI de redirection autorisés : https://[VOTRE_PROJECT_REF].supabase.co/auth/v1/callback
💡 Le [VOTRE_PROJECT_REF] se trouve dans Supabase → Settings → General → Reference ID
4
Copier Client ID + Secret dans Supabase
Supabase Dashboard → Authentication → Providers → Google → toggle ON → coller :
Client ID : XXXXXX.apps.googleusercontent.com Client Secret : GOCSPX-XXXXXXXXXXXXXXXX
5
Mettre à jour le Client ID dans index.html
Chercher clientId: dans NEXUS_CONFIG.google (ligne ~1909) et remplacer par votre vrai Client ID :
google: { clientId: "VOTRE_CLIENT_ID.apps.googleusercontent.com", enabled: true, },
⚫ Partie B — GitHub OAuth
1
Créer une GitHub OAuth App
GitHub → Settings → Developer Settings → OAuth Apps → New OAuth App.
🔗 GitHub OAuth Apps
2
Remplir le formulaire
Application name : NEXUS Market Homepage URL : https://nexus.sn Authorization callback URL : https://[VOTRE_PROJECT_REF].supabase.co/auth/v1/callback
3
Copier Client ID + Secret dans Supabase
Supabase Dashboard → Authentication → Providers → GitHub → toggle ON → coller le Client ID et le Client Secret générés.
✅ Une fois les deux providers activés dans Supabase, les boutons "Continuer avec Google" et "Continuer avec GitHub" s'affichent automatiquement sur la page de connexion NEXUS — le code est déjà là.
