Checklist avant lancement — NEXUS Market
0 / 22
🚨 Bloquants — ne pas lancer sans ces éléments
Intégrer un vrai paiement Mobile Money
La route /api/payments/mobile-money est une simulation (génère un faux ref). Intégrer PayTech Sénégal, CinetPay ou l'API Wave/Orange Money réelle.
Critique
Passer Stripe en mode production
Les clés actuelles sont sk_test_ / pk_test_. Remplacer par des clés live dans Railway + Vercel. Tester le webhook Stripe en production.
Critique
Créer les tables Supabase manquantes
Tables définies en commentaire SQL dans server.js mais pas forcément créées : refresh_tokens, coupons, loyalty_points, referrals, payout_requests, buyer_pro_profiles.
Critique
Configurer toutes les variables d'environnement
Manquants probables : API_URL, VAPID_PUBLIC_KEY, VAPID_PRIVATE_KEY, GITHUB_CLIENT_ID, GITHUB_CLIENT_SECRET, RESEND_API_KEY, SENTRY_DSN, STRIPE_WEBHOOK_SECRET.
Critique
Tester le flux complet de commande en prod
Parcours : inscription → produit → panier → checkout → paiement → confirmation email → statut vendeur → livraison. À tester avec un vrai compte sur l'URL de production.
Critique
Vérifier que les emails transactionnels partent
Confirmation de commande, réinitialisation de mot de passe, confirmation email inscription. Tester avec Resend ou SMTP configuré — pas EmailJS seul (EmailJS a des limites).
Critique
⚠️ Importants — à finir avant J-1
Page de suivi de commande acheteur
Le champ tracking_number existe dans la DB mais il n'y a pas d'UI de suivi visible. L'acheteur doit pouvoir suivre sa livraison depuis son compte.
Important
Page Fidélité (points) complète
awardLoyaltyPoints() existe côté serveur et la notification renvoie vers /loyalty. Mais la page UI complète (solde, historique, utilisation) n'est pas visible dans le code.
Important
Remplacer les icônes PWA placehold.co
Le manifest.json utilise https://placehold.co/192x192 et 512x512 comme icônes d'app. Créer de vraies icônes PNG branded NEXUS Market.
Important
Remplacer l'OG image placehold.co
La meta og:image pointe vers placehold.co/1200x630. Créer une vraie image de partage branded (affichée sur WhatsApp, Facebook, etc.).
Important
Configurer les push notifications VAPID
Le code web-push est complet mais nécessite VAPID_PUBLIC_KEY et VAPID_PRIVATE_KEY dans Railway. Sans elles, les notifications ne partent pas.
Important
Vérifier l'approbation vendeur côté admin
Le workflow de validation vendeur (pending → approved) doit être testé : admin reçoit-il une notif ? Le vendeur reçoit-il un email de confirmation ?
Important
Tester le système de parrainage complet
Le code backend insert dans referrals mais l'UI pour partager son code et voir ses filleuls n'est pas clairement visible. À vérifier et compléter.
Important
Configurer un cron externe de keep-alive
Créer un job sur cron-job.org ou UptimeRobot pour pinger /api/health toutes les 10 min — en complément du self-ping serveur déjà ajouté.
Important
🔵 Légal & Confiance
Contenu juridique réel dans les CGU
Les modales CGU et Politique de confidentialité existent. Vérifier que le contenu est rédigé par un juriste ou adapté au droit sénégalais (loi 2008-12).
Légal
Page Contact / À propos / FAQ
Les acheteurs et vendeurs doivent pouvoir contacter le support (email, WhatsApp). Une FAQ réduit les demandes répétitives avant lancement.
Légal
