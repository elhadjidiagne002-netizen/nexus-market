Récupérer vos
clés API Supabase

Ces clés permettent à votre application de se connecter à la base de données. Saisissez-les ici — elles seront utilisées automatiquement dans toutes les étapes suivantes.

🔑

C'est quoi une clé API ?

Une clé API est comme un badge d'accès. Elle prouve à Supabase que c'est bien votre application qui essaie d'accéder à vos données. Sans elle, personne ne peut entrer.

1 Trouver les clés dans Supabase
1

Ouvrir les paramètres

Votre projet Supabase → menu gauche → icône ⚙️ Settings → API

2

Repérer les 3 valeurs

• Project URL : commence par https://xxxxxxxxxxxx.supabase.co
• anon public : longue chaîne commençant par eyJhbGci...
• service_role : longue chaîne commençant par eyJhbGci... (encore plus longue)

🔑 Entrez vos 3 clés ici
⚡
Ces valeurs sont automatiquement propagées dans tous les blocs de code et variables d'environnement du guide. Une seule saisie suffit.
Settings → API → "Project URL"
Settings → API → "anon public" — peut être visible dans index.html
Settings → API → "service_role" — jamais dans index.html, uniquement dans .env et Vercel

Étape 6 / 22 ⏱ 5 min

Configurer la
sécurité des données

Les politiques RLS (Row Level Security) contrôlent qui peut voir quoi dans votre base de données. Sans elles, n'importe qui pourrait lire toutes vos données.

🔐

C'est quoi RLS ?

Imaginez que chaque ligne de votre base de données a un cadenas. RLS définit qui a la clé. Par exemple : un acheteur voit ses propres commandes mais pas celles des autres.

⚠️
Étape critique de sécurité. Sans ces politiques, votre base de données est ouverte à tous. Ne passez pas cette étape.
1 Activer RLS sur toutes les tables
1

SQL Editor → New query

Même méthode qu'à l'étape précédente : SQL Editor → + New query

2

Coller et exécuter

Copiez le bloc ci-dessous et cliquez "Run"

SQL — Politiques de sécurité
-- Activer RLS sur toutes les tables alter table profiles enable row level security; alter table products enable row level security; alter table orders enable row level security; alter table messages enable row level security; alter table reviews enable row level security; alter table notifications enable row level security; alter table email_logs enable row level security; -- Profiles : chacun voit son propre profil create policy "profiles_select" on profiles for select using (auth.uid() = id); create policy "profiles_update" on profiles for update using (auth.uid() = id); create policy "profiles_insert" on profiles for insert with check (auth.uid() = id); -- Products : lecture publique, modification vendeur uniquement create policy "products_public_read" on products for select using (status = 'active'); create policy "products_vendor_write" on products for all using (auth.uid() = vendor_id); -- Orders : acheteur et vendeur concernés uniquement create policy "orders_parties" on orders for select using (auth.uid() = buyer_id or auth.uid() = vendor_id); create policy "orders_insert" on orders for insert with check (auth.uid() = buyer_id); -- Messages : expéditeur et destinataire uniquement create policy "messages_parties" on messages for select using (auth.uid() = sender_id or auth.uid() = receiver_id); create policy "messages_insert" on messages for insert with check (auth.uid() = sender_id); -- Notifications : utilisateur concerné uniquement create policy "notifs_owner" on notifications for all using (auth.uid() = user_id); -- Reviews : lecture publique, écriture par acheteur create policy "reviews_read" on reviews for select using (true); create policy "reviews_write" on reviews for insert with check (auth.uid() = buyer_id); -- ✅ Sécurité configurée !

Configurer le
stockage des images

Les vendeurs doivent pouvoir uploader des photos de leurs produits. Supabase Storage est comme un disque dur en ligne gratuit pour vos images.

🖼️

Pourquoi un stockage séparé ?

Les images ne se stockent pas dans une base de données (trop lourd). On utilise un service spécialisé comme Supabase Storage qui optimise l'affichage des images dans le monde entier.

1 Créer les buckets via l'interface
1

Aller dans Storage

Supabase → menu gauche → 🗂️ Storage → "New bucket"

2

Créer le bucket "products"

Name: products → cocher "Public bucket" → Create bucket

3

Créer le bucket "avatars"

Name: avatars → cocher "Public bucket" → Create bucket

2 Configurer les permissions (SQL)
-- Politique storage : produits create policy "products_public_read" on storage.objects for select using (bucket_id = 'products'); create policy "products_vendor_upload" on storage.objects for insert with check (bucket_id = 'products' and auth.role() = 'authenticated'); create policy "products_vendor_delete" on storage.objects for delete using (bucket_id = 'products' and auth.uid()::text = (storage.foldername(name))[1]); -- Politique storage : avatars create policy "avatars_public_read" on storage.objects for select using (bucket_id = 'avatars'); create policy "avatars_owner_upload" on storage.objects for insert with check (bucket_id = 'avatars' and auth.role() = 'authenticated'); -- ✅ Stockage configuré !

🔧 Erreurs fréquentes
❌ "Build failed" — SUPABASE_URL invalid▼
Cause : Variable d'environnement manquante ou mal copiée.
Solution : Vercel → Settings → Environment Variables → vérifiez NEXT_PUBLIC_SUPABASE_URL (pas d'espace, pas de guillemets)
❌ "Module not found"▼
Cause : Le fichier package.json n'est pas à la racine du dépôt GitHub.
Solution : Vérifiez que tous vos fichiers sont bien à la racine, pas dans un sous-dossier.
⚠️ Site en ligne mais page blanche▼
Cause : Erreur JavaScript côté client. Ouvrez la console du navigateur (F12 → Console).
Solution : Souvent une variable d'environnement NEXT_PUBLIC_ manquante. Ajoutez-la et redéployez.

💳 Phase 5 — Stripe
Étape 17 / 22 ⏱ 8 min

Configurer
les paiements Stripe

Stripe gère les paiements par carte bancaire de façon sécurisée. L'argent est transféré directement sur votre compte bancaire.

💳

Comment Stripe fonctionne ?

Quand un client paie, sa carte bancaire est débitée par Stripe (qui est certifié PCI-DSS, le standard de sécurité bancaire mondial). Stripe vous reverse les fonds, moins une commission de 1,5% + 0,25€ par transaction.

1 Créer votre compte Stripe
1

Aller sur Stripe

Ouvrez stripe.com → "Start now"

2

Créer un compte

Email + mot de passe → vérifiez votre email

3

Compléter votre profil

Stripe demande des informations sur votre activité (type de commerce, pays, etc.) pour activer les paiements réels

4

Récupérer vos clés API

Dashboard Stripe → Developers → API Keys → copiez la Publishable key et la Secret key

💡
Commencez en mode test (clés commençant par pk_test_ et sk_test_). Vous pourrez passer en mode live quand tout fonctionne. En mode test, aucun vrai argent ne circule.

💳 Phase 5 — Stripe
Étape 18 / 22 ⏱ 6 min

Configurer les
Webhooks Stripe

Les webhooks permettent à Stripe de notifier votre application quand un paiement réussit ou échoue, pour mettre à jour les commandes automatiquement.

🔔

C'est quoi un webhook ?

C'est comme une sonnette. Quand Stripe traite un paiement, il "sonne" votre site en envoyant des données à une URL spéciale. Votre site écoute et met à jour la commande en conséquence.

1 Créer le webhook Stripe
1

Aller dans Webhooks

Stripe Dashboard → Developers → Webhooks → "Add endpoint"

2

URL de l'endpoint

Entrez cette URL (auto-générée depuis votre déploiement) :

https://votre-site.vercel.app/api/webhooks/stripe
3

Sélectionner les événements

Cliquez "Select events" → cochez :
payment_intent.succeeded
payment_intent.payment_failed
checkout.session.completed

4

Copier le Signing Secret

Après création → cliquez sur le webhook → "Reveal" → copiez le whsec_...

À ajouter dans les variables Vercel → Settings → Environment Variables
⚠️
Après avoir ajouté le Webhook Secret dans Vercel, redéployez votre application pour que la variable soit prise en compte : Vercel → votre projet → Deployments → bouton ⋮ → "Redeploy".

Ajouter un
domaine personnalisé

Optionnel mais professionnel : au lieu de nexus-market.vercel.app, votre marché sera accessible via votremarche.com. Cela inspire confiance et facilite le référencement.

🌐

C'est quoi un domaine ?

Un domaine est votre adresse sur Internet (comme votremarche.com). Vous le louez chez un "registrar" (OVH, Gandi, Namecheap...) pour ~10-15€/an, puis vous le configurez pour pointer vers Vercel.

ℹ️
Cette étape est optionnelle. Vous pouvez sauter et utiliser l'URL *.vercel.app pour commencer, puis ajouter un domaine plus tard.
🌐 Ajouter votre domaine à Vercel
1

Acheter un domaine

Si vous n'en avez pas : OVH, Namecheap ou Gandi — cherchez votre nom → achetez (~10€/an)

2

Ajouter dans Vercel

Vercel → votre projet → Settings → Domains → "Add" → tapez votremarche.com

3

Configurer les DNS

Vercel affiche 2 valeurs DNS. Allez chez votre registrar → Zone DNS → ajoutez un enregistrement A pointant vers l'IP Vercel et un CNAME pour www

4

Mettre à jour NEXT_PUBLIC_APP_URL

Vercel → Settings → Environment Variables → mettez à jour NEXT_PUBLIC_APP_URL avec votre nouveau domaine → Redeploy

5

Attendre la propagation

La propagation DNS prend entre 15 minutes et 48 heures. Vercel active le HTTPS automatiquement.
